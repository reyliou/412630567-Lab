# 霓虹計畫 (Project Neon) - CTF 實驗室說明文件

歡迎來到「霓虹計畫」滲透測試實驗室。這是一個基於真實漏洞設計的多階段 CTF 環境，旨在考驗您對現代 Web 應用程式、API 安全以及底層系統架構的分析能力。

## 🌐 實驗室存取資訊

*   **入口網址**：`http://<伺服器_IP>:8080`
*   **預設網域名稱**：`neo-mall.local` (可自行於 /etc/hosts 設定)
*   **開放服務**：
    *   **Port 8080**：Nginx Gateway (主要入口)
    *   **Port 21**：內部備份 FTP 伺服器 (僅供內網或特定情境存取)

## 🎯 您的任務

您的目標是從外部黑箱測試開始，逐步獲取系統權限，最終取得位於系統深處的 **Root Flag**。

### 階段目標：
1.  **身分識別**：尋找進入系統的憑據。
2.  **權限提升**：從一般用戶提升至賣家 (Seller) 權限。
3.  **系統滲透**：利用已知漏洞存取伺服器內部檔案。
4.  **終極目標**：獲取 `/root/root_flag.txt`。

## 🚩 Flag 說明

*   **Flag 格式**：`FLAG{學號_隨機雜湊}`
*   **動態生成**：本環境具備動態 Flag 生成機制。在滲透過程中，您可能需要觸發特定的「系統診斷」腳本來生成屬於您的專屬 Flag。
*   **範例**：`FLAG{412630567_d41d8cd98f00b204e9800998ecf8427e}`

## 💡 入門提示 (Hints)

1.  **魔鬼隱藏在細節中**：前端頁面看不到的東西，API 或許會告訴你答案。試著對產品 API 進行列舉 (`Enumeration`)。
2.  **開發者的便利是攻擊者的捷徑**：尋找開發團隊留下的除錯介面或測試帳號。
3.  **模板的代價**：當系統允許你自訂內容並預覽時，注意它是否不安全地渲染了指令。
4.  **已知漏洞的威力**：系統的核心組件之一使用的是 `aiohttp 3.9.1`，這是一個具備公開 CVE 漏洞的版本。

## 🛠 建議工具

*   **瀏覽器開發者工具** (F12)
*   **Burp Suite** (用於攔截與修改 Request)
*   **cURL** (處理特殊的 HTTP 路徑跳轉)
*   **Netcat / Nmap** (基礎網路掃描)

## ⚠️ 實驗規則

*   請勿對伺服器進行大規模暴力破解或 DDoS 攻擊。
*   本環境僅供資安教育與期末專題使用，請勿將相關技術用於非授權場域。
